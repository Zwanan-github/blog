## 一、复杂度分析

- 大O表示法：描述算法运行时间/空间随输入规模增长的趋势（上界）
- 常见量级：O(1) < O(logn) < O(n) < O(nlogn) < O(n²) < O(2ⁿ)
- 均摊时间复杂度：动态数组扩容、栈的 push/pop 等场景

---

## 二、线性数据结构

### 数组
- 连续内存，随机访问 O(1)，插入/删除 O(n)
- 动态扩容：Java ArrayList 1.5倍，Go slice 容量翻倍（小容量）

### 链表（高频）
- 单链表 / 双链表 / 循环链表
- 高频操作：反转、判环（Floyd 快慢指针）、找中点、合并有序链表、K个一组翻转

### 栈
- LIFO；用数组或链表实现
- 应用：括号匹配、函数调用栈、**单调栈**（下一个更大元素、接雨水）

### 队列
- FIFO；循环数组实现避免假溢出
- 双端队列（Deque）：滑动窗口最大值
- 优先队列（堆实现）：TopK、任务调度

### 哈希表（高频）
- 哈希函数 + 冲突解决（链地址法 / 开放寻址法）
- Java HashMap：数组+链表+红黑树，负载因子0.75，扩容2倍，链表长度≥8转红黑树
- 高频问题：哈希冲突原理、扩容机制、ConcurrentHashMap 线程安全实现

## 三、树

### 二叉树（高频）
- 前/中/后序遍历（递归 & 迭代两种写法）、层序遍历（BFS）
- 高频题：最大深度、对称判断、最近公共祖先（LCA）、路径总和、序列化与反序列化

### 二叉搜索树（BST）
- 左 < 根 < 右，中序遍历有序
- 查找/插入/删除平均 O(logn)，最坏 O(n)（退化链表）

### 红黑树（高频）
- 近似平衡，插入/删除/查找均 O(logn)
- 5条性质：节点红/黑、根黑、叶NIL黑、红节点子节点黑、任意节点到叶黑节点数相同
- Java TreeMap、TreeSet、HashMap（链表转树）底层使用

### B+ 树（高频，MySQL索引）
- 非叶节点只存索引键，叶节点存数据且用双向链表相连
- 相比B树优势：叶节点链表支持范围查询；非叶节点更小，同层可缓存更多节点
- **MySQL InnoDB 索引为什么用B+树而不是红黑树**：B+树高度低（3层可存千万行），磁盘IO少

### 堆（高频）
- 完全二叉树，数组存储，父节点索引 i，子节点 2i+1 / 2i+2
- 大根堆 / 小根堆；插入/删除 O(logn)，建堆 O(n)
- 应用：优先队列、**TopK问题**（维护大小为K的堆）、堆排序

### 字典树（Trie）
- 多叉树，每个节点代表一个字符，用于前缀匹配、单词搜索

## 四、图

- 表示：邻接矩阵（稠密图）、邻接表（稀疏图）
- BFS：队列，求无权图最短路径、层序扩展
- DFS：栈/递归，求连通分量、拓扑排序、全路径
- 最短路径：Dijkstra（非负权，贪心+优先队列）、Bellman-Ford（含负权）
- 最小生成树：Prim（贪心）、Kruskal（并查集）
- 拓扑排序：Kahn算法（BFS）、DFS后序逆序

## 五、排序算法（高频）

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定 |
|------|---------|---------|------|------|
| 冒泡 | O(n²) | O(n²) | O(1) | ✅ |
| 插入 | O(n²) | O(n²) | O(1) | ✅ |
| 归并 | O(nlogn) | O(nlogn) | O(n) | ✅ |
| 快排 | O(nlogn) | O(n²) | O(logn) | ❌ |
| 堆排 | O(nlogn) | O(nlogn) | O(1) | ❌ |
| 计数 | O(n+k) | O(n+k) | O(k) | ✅ |

**快排核心**：partition 分区，随机选基准避免最坏情况，递归左右子数组  
**归并核心**：分治，先分后合，适合链表排序（不需要随机访问）  
**面试常问**：为什么快排平均比堆排快？（缓存友好，局部性好）

## 六、常用算法思想

### 双指针
- 对撞指针：有序数组两数之和、三数之和
- 快慢指针：链表判环、找中点、删除倒数第K个节点
- 滑动窗口：最长无重复子串、最小覆盖子串、固定窗口求最大值

### 二分查找（高频）
- 前提：有序或具有单调性
- 模板：`left <= right`，`mid = left + (right-left)/2` 防溢出
- 变体：找第一个/最后一个满足条件的位置（左边界/右边界二分）

### 递归与回溯
- 回溯 = 递归 + 撤销，用于排列/组合/子集/N皇后/数独
- 剪枝是关键：提前终止无效分支

### 动态规划（高频）
- 核心：最优子结构 + 重叠子问题
- 步骤：定义状态 → 状态转移方程 → 初始值 → 遍历顺序
- 高频题型：
  - 背包：0-1背包、完全背包
  - 序列：LCS（最长公共子序列）、LIS（最长递增子序列）、编辑距离
  - 路径：最小路径和、不同路径
  - 打家劫舍系列、股票买卖系列

### 贪心
- 每步选局部最优推导全局最优（需证明贪心选择性质）
- 典型：区间调度（按结束时间排序）、跳跃游戏、分发糖果

### 并查集
- 支持合并（union）和查找（find）操作，路径压缩+按秩合并近似O(1)
- 应用：连通分量、最小生成树Kruskal、岛屿数量

## 七、高频面试题分类

### 链表
- 反转链表（迭代/递归）

- 二叉树层序遍历（BFS）

- 最长无重复子串
- 最小覆盖子串
- 滑动窗口最大值
